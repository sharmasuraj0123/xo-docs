XO's **One-Click Deploy** button does more than put your app online — it spins up a complete CI/CD environment with real infrastructure behind the scenes.

## What Happens When You Click "Deploy"

<Steps>
  <Step>
    ### Provision infrastructure via Kubernetes on AWS

    A Kubernetes-managed pod is spun up dynamically to run your app, ensuring scalability and containerised isolation.
  </Step>

  <Step>
    ### Install required dependencies

    XO auto-detects your tech stack (Node.js, Python, Solidity) and installs necessary packages automatically.
  </Step>

  <Step>
    ### Set up a full CI/CD pipeline

    Your app is versioned, built, and containerised. Build artifacts are tested, validated, and deployed. Logs and errors are streamed in real-time.
  </Step>

  <Step>
    ### Deploy the live app to your personal URL

    Your Kubernetes pod is exposed via a live HTTPS endpoint with custom domain support coming soon.
  </Step>

  <Step>
    ### Monitor and manage the deployment

    With Kubernetes-native monitoring and restart logic, your app stays resilient and fault-tolerant.
  </Step>
</Steps>

## Behind the Scenes

The deployment architecture includes:

- Kubernetes-powered autoscaling clusters for reliable, containerised app hosting
- Git-like versioning and rollback support for all deployments
- Dynamic environment setup using Helm charts and Docker containers
- Nginx ingress controllers for routing traffic
- Real-time CI/CD logging dashboard integrated into the XO interface
- AWS-backed compute running on EC2 inside a managed K8s cluster

## Who It's For

- **Hackers** who want to launch demos or test apps without manual DevOps
- **Founders** who need fast MVP launches with integrated infra and CI/CD
- **Teams** building reproducible, isolated staging or production environments
