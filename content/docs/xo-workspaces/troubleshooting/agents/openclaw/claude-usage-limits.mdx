## What Happened

Anthropic changed how Claude subscriptions work. Claude Pro and Team plans no longer cover API usage in third-party tools like OpenClaw. Your agent didn't break — it needs a new auth method to keep running.

If your agent replied with this on Telegram, WhatsApp, or any other channel:

<Callout type="warn">
  LLM request rejected: Third-party apps now draw from your extra usage, not your plan limits. We've added a $x credit to get you started. Claim it at claude.ai/settings/usage and keep going.
</Callout>

Fix it by reconfiguring your model provider inside your workspace. The whole process takes under 5 minutes.

## Fix It

<Steps>
<Step>
### Open your project

Go to [beta.xo.builders](https://beta.xo.builders), find the project showing the error, and open the [Configure panel](/docs/xo-workspaces/core-platform-setup/access-restart-machine).

![Configure button](/images/troubleshooting/configure.jpeg)
</Step>

<Step>
### Open your workspace IDE

Click **Open Workspace** and choose your IDE — VS Code, Cursor, or Code Server (browser-based VS Code). All three work the same way for this.

![Open workspace IDE options](/images/troubleshooting/cursor-desktop.jpeg)
</Step>

<Step>
### Run the config tool

Open your workspace from the [Configure panel](/docs/xo-workspaces/core-platform-setup/access-restart-machine#opening-your-ide), [open a terminal](/docs/xo-workspaces/core-platform-setup/vs-code-server#open-a-terminal), and run:

```bash
openclaw config
```
</Step>

<Step>
### Select "Local" for the gateway location

Keep **Local** selected and hit **Enter**.

![Select Local gateway](/images/troubleshooting/openclaw-configure-local.png)
</Step>

<Step>
### Select "Model" from the sections list

Select **Model** and hit **Enter**.

![Select Model section](/images/troubleshooting/model.png)
</Step>

<Step>
### Choose your provider and enter credentials

Pick the provider you want to use. Full instructions for each option are below.

![Model/auth provider list](/images/troubleshooting/select-provider.png)

<Tabs items={['Anthropic API Key', 'Other Providers', 'Codex']}>
  <Tab value="Anthropic API Key">
    Select **Anthropic** from the provider list, then select **Anthropic API key** and hit **Enter**.

    ![Select Anthropic API key auth method](/images/troubleshooting/select-method.png)

    If prompted about an existing key, select **No** to enter a new one.

    ![Select No to use existing key](/images/troubleshooting/select-no-to-prior-key.png)

    Paste your API key and hit **Enter**. Get one at [console.anthropic.com](https://console.anthropic.com) if you don't have one yet.

    ![Enter Anthropic API key](/images/troubleshooting/enter-anthropic.png)

    <Callout type="info">
      Newer versions of OpenClaw may show a slightly different auth menu, but the **Anthropic API key** option will always be present.
    </Callout>
  </Tab>

  <Tab value="Other Providers">
    Select any provider from the list — OpenAI, Google, DeepSeek, Mistral, and others are all supported. After selecting your provider, the terminal walks you through entering your credentials for that provider.

    The steps are the same pattern: select provider → select auth method (if applicable) → paste your API key.

    Get your API key from your chosen provider's developer console before starting.
  </Tab>

  <Tab value="Codex">
    Switching to Codex requires a few additional steps. Follow the [Switch to Codex](/docs/xo-workspaces/agents/openclaw/models/change-model-codex) guide.
  </Tab>
</Tabs>
</Step>

<Step>
### Restart the gateway

Once config is saved, restart OpenClaw to apply the changes. Open the [Configure panel](/docs/xo-workspaces/core-platform-setup/access-restart-machine) and click **OpenClaw Restart**, or run this in the terminal:

```bash
./xo-cowork-api/openclaw.sh restart
```

Wait a minute, then send a message to your agent to confirm it's responding again.
</Step>
</Steps>
