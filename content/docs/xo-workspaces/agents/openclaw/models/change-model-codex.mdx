## What You'll Need

- A [deployed OpenClaw agent](/docs/xo-workspaces/agents/openclaw)
- An OpenAI account (email, Google, or Apple login)

## Change the Model

<Steps>
  <Step>
    ### Open Your Project

    Go to [beta.xo.builders](https://beta.xo.builders) and select your OpenClaw project from the sidebar.

    ![Select your OpenClaw project on the dashboard](/images/codex-select-project-1.jpeg)
  </Step>
  <Step>
    ### Open Your Workspace

    Open your workspace from the [Configure panel](/docs/xo-workspaces/core-platform-setup/access-restart-machine#opening-your-ide), then [open a terminal](/docs/xo-workspaces/core-platform-setup/vs-code-server#open-a-terminal) and run:

    ```bash
    openclaw config
    ```

    ![Running openclaw configure in the terminal](/images/codex-openclaw-configure-1.png)
  </Step>
  <Step>
    ### Select Gateway Location

    When prompted with **"Where will the Gateway run?"**, select **Local (this machine)**.

    ![Selecting Local gateway location](/images/codex-select-local-2.png)
  </Step>
  <Step>
    ### Pick the Model Section

    The wizard asks which sections to configure. Use arrow keys and spacebar to select:

    - **Model** (pick provider + credentials)

    ![Selecting the Model section to configure](/images/codex-select-model-3.png)
  </Step>
  <Step>
    ### Choose OpenAI Codex

    In the **Model/auth provider** list, scroll down and select **OpenAI Codex**.

    ![Selecting OpenAI Codex as the model provider](/images/codex-select-openai-codex-4.png)
  </Step>
  <Step>
    ### Authenticate with OpenAI

    The terminal generates a unique OAuth URL and waits for a redirect URI. Copy the URL and open it in your local browser.

    ![OAuth URL ready in the terminal](/images/codex-auth-url-5.png)

    Sign in to OpenAI with your email, Google, Apple, or Microsoft account.

    ![OpenAI sign-in page](/images/codex-open-auth-browser-6.png)

    Review the **"Sign in to Codex with ChatGPT"** permissions screen and click **Continue**.

    ![Codex permissions screen — click Continue](/images/codex-sign-in-7.png)
  </Step>
  <Step>
    ### Complete the Connection

    After authentication, the browser redirects to a `localhost` URL. The page itself won't load — that's expected. Copy the full URL from the address bar.

    ![Copy the localhost redirect URI from the address bar](/images/codex-copy-uri-9.png)

    Switch back to your terminal and paste the redirect URI when prompted.

    ![Pasting the redirect URI in the terminal](/images/codex-paste-uri-10.png)
  </Step>
  <Step>
    ### Select Models

    The wizard shows the available Codex models. Use arrow keys and spacebar to pick the models you want in the `/model` picker (e.g., **gpt-5.3-codex**, **gpt-5.4**).

    ![Selecting models in the multi-select picker](/images/codex-select-model-11.png)
  </Step>
  <Step>
    ### Finish the Wizard

    Back at the section menu, scroll to **Continue (Done)** and press Enter. The config wizard finalizes and your agent now uses OpenAI Codex.

    ![Selecting Continue to finish configuration](/images/codex-continue-12.png)
  </Step>
</Steps>

## Verify the Config

Run `openclaw models list` in your terminal to confirm the available models, or open `openclaw.json` in your workspace. The `agents.defaults` section reflects the new model:

```jsonc
"agents": {
  "defaults": {
    "model": {
      "primary": "openai-codex/gpt-5.3-codex"
    },
    "models": {
      "openai-codex/gpt-5.3-codex": {},
      "openai-codex/gpt-5-2-codex": {},
      "anthropic/claude-opus-4-5": {}
    },
    "maxConcurrent": 4,
    "subagents": {
      "maxConcurrent": 8
    }
  }
}
```
