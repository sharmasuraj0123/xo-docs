XO Launchpad makes it simple to deploy a GaiaNet Node with just a few clicks. No manual configuration needed — just name your project and launch.

## What is GaiaNet?

GaiaNet is a decentralised AI compute and hosting network. Running a GaiaNet node lets you:

- Participate in the GaiaNet ecosystem
- Contribute compute resources
- Host AI-powered chatbots and agents

## Deploy Your Node

<Steps>
  <Step>
    ### Sign in to XO Launchpad

    Go to [launchpad.xo.builders](https://launchpad.xo.builders) and sign in.
  </Step>

  <Step>
    ### Open Templates

    From the sidebar, click **Templates**.
  </Step>

  <Step>
    ### Select GaiaNet Node

    Click the **GaiaNet Node** template.
  </Step>

  <Step>
    ### Name Your Project

    Provide a project name using lowercase letters, numbers, and hyphens only.

    Example: `gaianet-node-01`
  </Step>

  <Step>
    ### Create Deployment

    Click **Create Deployment**. XO Launchpad provisions your node automatically — no Dockerfiles or environment variables required.
  </Step>
</Steps>

<Callout type="info">
  Within minutes your node will be live and accessible via the provided URL.
</Callout>

## What Happens Next

- XO provisions the GaiaNet Node automatically
- No extra setup required
- Your node is live and accessible via a generated URL

Learn more at [GaiaNet Docs](https://docs.gaianet.ai) or explore more templates like the [N8N Workflows](/docs/xo-launchpad/templates/n8n-workflows).
