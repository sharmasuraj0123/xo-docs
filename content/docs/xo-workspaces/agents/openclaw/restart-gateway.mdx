Two ways to restart your OpenClaw gateway — the dashboard button (fastest) or the terminal command (gives you logs).

<Tabs items={['Dashboard Button', 'Terminal Command']}>
  <Tab value="Dashboard Button">
    ## Restart from Dashboard

    1. Open your project on [beta.xo.builders](https://beta.xo.builders)

    ![Project page](/images/restart-project-page.jpeg)

    2. Click the **OpenClaw Restart** button

    ![Click OpenClaw Restart](/images/restart-click-button.jpeg)

    3. Wait for the confirmation message

    ![Gateway restarted](/images/restart-confirmed.jpeg)

    The gateway is now restarting. Wait a minute before opening the gateway again.
  </Tab>

  <Tab value="Terminal Command">
    ## Restart from Terminal

    1. Click **Configure** on your project

    ![Click Configure](/images/restart-click-configure.jpeg)

    2. Open **code-server** (or any IDE)

    ![Click code-server](/images/restart-click-codeserver.jpeg)

    3. Open the terminal with `Ctrl + backtick`

    ![Open terminal](/images/restart-open-terminal.jpeg)

    4. Run the restart command:

    ```bash
    ./xo-cowork-api/openclaw.sh restart
    ```

    ![Run restart command](/images/restart-run-command.jpeg)

    5. Check the output for any errors

    ![Gateway output](/images/restart-gateway-output.jpeg)

    6. Check `openclaw-gateway.log` for detailed logs if needed

    ![Gateway log file](/images/restart-gateway-log.jpeg)

    <Callout type="info">
      The terminal method is better for debugging — you can see errors in real time and check log files.
    </Callout>
  </Tab>
</Tabs>
